Project Structure
=========================================================================================================
fastran_project/
│
├── fastran_gui.py          # The main application script (formerly the large v2.3.4)
├── parsers.py              # Handles reading/writing .fin and material files
├── editors.py              # Contains all Toplevel editor windows (Spectrum, Block, Dkeff, etc.)
├── widgets.py              # Contains reusable UI components (HelpWindow, ToolTip, etc.)
├── config.py               # NEW: Holds all mapping dicts and default values (renamed from utils.py)
├── utils.py                # NEW: Holds simple helper functions (safe_float, etc.)
├── runners.py               # NEW: Manages running fastran.exe and dkeff.exe
│
├── fastran.exe             # The FASTRAN executable
├── dkeff.exe               # The dkeff utility executable
└── fastran_gui.cfg         # The configuration file for exe paths

Consolidated Roadmap
=========================================================================================================
### ⚡ Optimization / Code Quality
Validation Layer: Create a dedicated validation function (e.g., in utils.py or a new validator.py) that checks all GUI inputs before a run. It would verify that CI < CF, all required numeric fields are valid, etc., and present a clear list of errors to the user if any are found.

Centralized Logging: Implement Python's built-in logging module. Configure it to write all events (info, warnings, errors) to the output/log.txt file within the project. This is invaluable for debugging user issues.

### 🆕 New Features
Parameter Study / Batch Processing: Create a new "Parameter Study" editor window. This would allow a user to select a parameter (like CI), define a range (e.g., from 0.1 to 0.5 in steps of 0.1), and have the application automatically run an analysis for each value, saving each result in a uniquely named subfolder in the output/ directory.

Material Library / Database: Enhance the application to manage a library of materials. Users could save the properties from "Tab 1" as a named material (e.g., "7075-T6"). A new materials/ folder in the project could store these as simple .json files, which could then be selected from a dropdown menu.

Project Templates: Create a templates/ folder with pre-filled .frproj folders for common analysis types (e.g., "Compact Tension Specimen," "Center Crack Panel"). Add a File > New From Template... menu option to let users create new projects based on these.

### 🎨 User Interface (UX) Improvements
Interactive Plotting: In the PostProcessingWindow, add the ability to load and overlay multiple result sets on the same plot for easy comparison. Add tooltips that show the exact X/Y values when a user hovers over a data point on the graph.

Context-Sensitive Help: Add small (?) icons next to key input fields (like NTYP, NALP, NFOPT). When clicked, they would open the Help Window directly to the relevant section.

Auto-Save & Recovery: As a more advanced feature, have the application periodically save the state of the GUI to a temporary file. If the app closes unexpectedly, it can offer to restore the unsaved session on the next launch.
